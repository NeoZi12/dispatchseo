---
title: Install on a VPS
description: One line installs everything - Docker included - and one .env line puts the dashboard on your own domain with automatic HTTPS.
---

Any provider works (Hetzner, DigitalOcean, Vultr, ...); a 1 GB Ubuntu or
Debian box is enough. About five minutes end to end.

<Steps>

<Step title="Connect to the server">

From your own terminal:

```bash
ssh root@your-server-ip
```

</Step>

<Step title="Install and start">

One line - it installs Docker if the server doesn't have it, fetches
DispatchSEO, and boots the stack:

```bash
curl -fsSL https://dispatchseo.com/install.sh | sh
```

Re-running the same line later is also the upgrade command. (Prefer to do
it by hand? `git clone https://github.com/NeoZi12/dispatchseo && cd
dispatchseo && sh start.sh` is all the script does.)

When it finishes, the stack is listening on port 4005 of the server - but
only locally. Don't open that port to the internet; the next step gives
the dashboard a proper HTTPS address instead.

</Step>

<Step title="Give it its address: your domain, automatic HTTPS">

You already own a domain - your website's - so this is one DNS record and
one line in `.env`. Never touched DNS before? The record part, spelled
out:

**a. Open your domain's DNS settings.** They live wherever your domain is
managed - usually the registrar you bought it from (Namecheap, GoDaddy,
Porkbun, ...), or Cloudflare / Vercel if you've pointed the domain's
nameservers there. Log in, pick the domain, and look for the section
called **DNS**, **DNS records**, **Manage DNS**, or **Zone editor** -
every host has one, the name just varies.

**b. Add one A record.** There's an "Add record" button; fill the form
with exactly this:

| Field | Paste / pick exactly this |
| --- | --- |
| Type | `A` |
| Name (or Host) | `dispatch` |
| Value (or "Points to") | your server's IP - the same one you SSH into, shown in your VPS provider's dashboard |
| TTL | leave whatever it already says |

That creates `dispatch.your-domain.com`, and it's the address every
command below uses. Save the record. **On Cloudflare:** switch the record to **DNS only** (the grey
cloud, not the orange "Proxied" one) - the proxy blocks the certificate
handshake in the next step.

**c. Confirm it landed.** After a few minutes, run this on your own
computer - it should print your server's IP:

```bash
ping -c 1 dispatch.your-domain.com
```

**d. Tell DispatchSEO the domain** - swap in yours:

```bash
cd dispatchseo &&
  echo "DOMAIN=dispatch.your-domain.com" >> .env &&
  sh start.sh
```

That's it. The stack now includes its own HTTPS proxy: it fetches the
certificate, renews it forever, and `start.sh` prints your new address.
Open `https://dispatch.your-domain.com`, choose a dashboard password, and
the setup wizard starts. From now on the dashboard is a bookmark, from
any device.

Two notes for unusual setups:

- **Provider firewall:** if your VPS has a cloud firewall (Hetzner Cloud
  Firewall, DO Cloud Firewall, AWS security groups), allow ports **80 and
  443** in its panel. Plain `ufw` boxes need nothing - Docker publishes
  the ports directly.
- **Already running Nginx/Traefik/Caddy on this host?** Leave `DOMAIN`
  unset (ports 80/443 stay yours), point your existing proxy at
  `localhost:4005`, and set `APP_URL=https://your-subdomain` in `.env`
  yourself.

(Waiting on DNS and impatient? `ssh -L 4005:localhost:4005
root@your-server-ip` on your own computer tunnels the dashboard to
[http://localhost:4005](http://localhost:4005), encrypted. You won't need
it after DNS lands.)

</Step>

<Step title="The builder token comes from your own computer">

`claude setup-token` opens a browser login, which a headless server can't
do. Run it on your own machine, copy the `sk-ant-oat...` token into the
VPS's `.env` as `CLAUDE_CODE_OAUTH_TOKEN=...`, then on the server:

```bash
docker compose up -d builder
```

That's the whole difference from a local install. The setup wizard and
everything after it are identical - except this machine never sleeps, so
the schedules fire at their scheduled times.

</Step>

</Steps>
